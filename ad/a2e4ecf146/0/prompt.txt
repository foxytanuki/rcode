# Refactor Plan: rcode コードベース整理

## Context

rcode は v0.2.x 台の初期開発フェーズ。機能追加に伴い以下の技術的負債が蓄積:
- `internal/editor/` パッケージを作ったが、サーバーが使っていない（独自の executor.go を持つ）
- Tailscale検出ロジックが2箇所に重複
- レガシー Config 型が新型と共存
- client.go 内のfallbackパターンが4回コピペ

このrefactorで重複を排除し、internal パッケージを正しく活用する状態にする。

## Step 1: サーバーExecutorを `internal/editor` に統合 (高優先度)

最大の構造的問題。`cmd/server/executor.go` (296行) が `internal/editor/` (956行) と完全に機能重複。

### 変更対象
- **削除:** `cmd/server/executor.go`
- **修正:** `cmd/server/server.go` — Server struct のフィールド変更
- **修正:** `cmd/server/handlers.go` — editor.Manager / editor.Executor 経由の呼び出しに変更

- **修正:** `cmd/server/executor_test.go` — internal/editor のテストに統合 or 削除

### 具体的な対応
1. `Server` struct を変更:
   ```go
   type Server struct {
       config    *config.ServerConfigFile
       log       *logger.Logger
       editor    *editor.Manager   // 旧: executor *Executor
       startTime time.Time
   }
   ```
2. `NewServer` で `editor.NewManager(cfg.Editors, log)` を呼ぶ
3. `handlers.go` の `handleOpenEditor` で `editor.Manager.GetEditor()` → `Template.Render()` → `editor.Executor.Execute()` に切り替え
4. `handleEditors` で `editor.Manager.ListEditors()` と `editor.Manager.IsAvailable()` を使う
5. `cmd/server/executor.go` の `init()` (PATH設定) は `cmd/server/main.go` に移動
6. `cmd/server/executor_test.go` は `internal/editor/` のテストでカバーされている部分を削除、不足があれば `internal/editor/` 側にテスト追加

## Step 5: Client の fallback パターン共通化 (低優先度)

### 変更対象
- **修正:** `cmd/rcode/client.go`

### 具体的な対応
`withFallback` ヘルパーを追加:
```go
func (c *Client) withFallback(fn func(host string) error) error {
    err := fn(c.config.Hosts.Server.Primary)
    if err == nil {
        return nil
    }
    c.log.Warn("Primary host failed", "host", c.config.Hosts.Server.Primary, "error", err)
    if c.config.Hosts.Server.Fallback != "" {
        if err2 := fn(c.config.Hosts.Server.Fallback); err2 == nil {
            return nil
        }
    }
    return fmt.Errorf("all hosts failed: %w", err)
}
```

`OpenEditor`, `ListEditors`, `fetchEditorCommand`, `CheckHealth` を書き換え。

ポート付与の共通化 (`ensurePort`) もここで一緒にやる。
